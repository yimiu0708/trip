# 识界 V0.4 需求文档

> 状态：已确认，作为 V0.4 开发与验收基准  
> 日期：2026-06-20

## 1. 版本目标

让用户从“记录过去的旅行”自然过渡到“收藏想去的地方，并获得下一次真实出行的轻量建议”。本版本不建设行程规划、实时天气交通、社区、排行榜、全球地图或大规模景区内容库。

## 2. 收藏闭环

- 支持收藏城市和已发布景区，不开放省份收藏。
- 城市内任一景区有点亮记录，即视为该城市已点亮；城市完整度另行展示。
- 收藏采用软删除，保留首次收藏、最近收藏、取消时间和首次“未点亮时收藏”时间。
- 取消收藏立即生效，并提供短时撤销。
- “从收藏到点亮”仅在目标收藏时尚未点亮、之后首次产生点亮记录时解锁。
- 我的收藏页路由为 `/favorites`，类型和点亮状态为两组独立筛选，排序支持未点亮优先、最近收藏和城市进度高。
- 【我的】三个主入口调整为：成就中心 / 旅行人格 / 我的收藏。足迹补录能力保留在地图、完成页和成就行动中。

## 3. 下一次出行推荐

- 接口：`GET /api/recommendations/next-trip?source=home|completion|favorites|achievement&limit=1..3`。
- 推荐目标只返回城市或景区；省份和片区规则命中后下钻为可收藏目标。
- 优先级：有效收藏 > 成就推进 > 快完成城市或省份 > 未触达片区 > 通用 5A。
- 快完成定义：已有足迹，且进度不低于 70%，或仅剩不超过 2 个目标。
- 每类先取最高分 1 张，再按优先级补满；同一目标在列表内只出现一次。
- 季节推荐只建议出行时间，不声称目的地具有未维护的季节适游属性。
- 游客和零数据用户展示按 5A 数量、景区总数、ID 稳定排序的通用推荐；游客收藏时引导登录。
- 首页展示 2 张，点亮完成页最多 3 张，收藏页排除当前有效收藏后补充推荐。接口失败不得影响页面主流程。
- 城市行动路由使用 `/province/:id/cities/:cityId`；景区增加 `?highlight=:attractionId`。

## 4. 成就扩展

### 四季旅人

- 春日漫游家、盛夏追光者、秋色收藏家、冬日远行者：对应季节点亮 3 个不同景区。
- 四季旅人：春夏秋冬均至少点亮 1 个景区。
- `time_precision` 为 `year` 或 `unknown` 的记录不参与；其他记录优先使用有效 `season`，否则按月份换算。

### 八方巡游

- 片区数据以后端 `provinces.region` 聚合为准，当前共 8 个片区。
- 任一省级行政区有景区足迹，即视为触达其所属片区。
- 等级条件：1 / 3 / 5 / 全部片区；片区卡同时展示省份覆盖进度。

### 收藏家

- 愿望清单：首次收藏城市或景区。
- 出发前收藏家：当前有效收藏中有 5 个未点亮目标。
- 从收藏到点亮：点亮 1 个收藏时尚未点亮的目标。
- 已解锁成就不回收；取消点亮或收藏只重算当前进度。

## 5. 数据与接口

- `user_favorites` 唯一键：`(user_id, target_type, target_id)`；目标类型仅允许 `city`、`attraction`。
- `GET /api/favorites` 返回统计、筛选排序后的有效收藏。
- `GET /api/favorites/keys` 返回当前有效收藏键，用于批量同步按钮状态。
- `POST /api/favorites` 幂等创建或恢复收藏；`DELETE /api/favorites/:id` 软删除。
- `GET /api/user/region-progress` 返回片区触达、省份覆盖与明细。
- `GET /api/achievements/mine` 保持数组响应兼容，新增 `group_key`、`subgroup_key` 和季节、片区、收藏进度字段。

## 6. 交互与验收

- 城市 Hero、景区行、推荐卡和收藏页均可收藏；景区勾选与收藏按钮点击互不影响。
- 所有心形按钮触控区域不小于 44px，状态具有可访问名称和键盘焦点。
- 推荐、收藏和成就页面延续当前蓝绿旅行视觉，收藏使用小面积暖橙提示。
- 新增 9 套独立 WebP 成就图稿：四季 5 套、八方巡游 1 套、收藏 3 套。
- 回归登录、地图、批量点亮、时间精度、多次点亮、旅程、旅行人格、原有成就和管理后台。
- 验证命令：`cd client && npm run build && npm run lint`、`cd server && npm run build`。
