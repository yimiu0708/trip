# 识界 · light your life —— UI 设计需求简报

> **用途**：交付专业 UI 设计团队，用于系统 UI 交互设计与视觉风格定义  
> **版本**：V0.1（MVP 完整功能版）  
> **日期**：2026年6月

---

## 一、品牌与产品定位

### 1.1 品牌信息

| 项目 | 内容 |
|------|------|
| **品牌名** | 识界（暂定，后续可能调整） |
| **Slogan** | light your life |
| **品牌理念** | 用旅行足迹丈量世界，每一次点亮都是对生命的注脚。鼓励用户以"地图点亮"的方式记录旅行、探索中国、可视化个人成就。 |
| **核心情绪** | 自由、探索、成就、温暖、记录 |

### 1.2 产品定位

一款面向国内旅行爱好者的轻量级足迹记录工具。核心以**"中国地图点亮"**为玩法载体，用户通过标记 4A/5A 级景区来可视化自己的旅行版图，配套成就系统激发持续探索欲望。

**一句话描述**：
> 在一张中国地图上，把你去过的每一个地方点亮成光。

### 1.3 目标用户画像

| 维度 | 描述 |
|------|------|
| **核心用户** | 25-45 岁国内旅行爱好者，年均出行 3-6 次 |
| **使用场景** | 旅行后记录足迹、回顾过往旅程、规划下一目的地、截图分享至社交平台 |
| **使用习惯** | 移动端为主，桌面端为辅；注重截图美观度 |
| **核心诉求** | ① 直观的可视化成就感 ② 简单的记录操作 ③ 好看的分享截图 |

### 1.4 MVP 边界（已实现）

- **包含**：地图展示、600+ 景区数据、点亮记录（支持多次+自定义日期）、分类管理、用户账号、成就系统、管理后台、移动端适配
- **排除**：社交分享（后续规划）、行程规划、UGC 内容、商业化功能

## 三、页面结构与功能清单

系统共 **7 个核心页面**，采用 **SPA 单页应用** 架构。

### 3.1 导航体系

#### 桌面端（> 768px）
- **顶部固定导航栏**（高 48px）：左侧品牌名"识界 · light your life"，右侧头像下拉菜单（个人资料 / 修改密码 / 管理后台 / 退出）
- **无底部 Tab**

#### 移动端（≤ 768px）
- **顶部固定导航栏**（高 48px）：仅左侧品牌名
- **底部固定 Tab 栏**（高 56px，含安全区适配）：3 个入口 — 地图 / 旅程 / 成就

### 3.2 页面清单

---

#### 页面 1：地图首页（`/` 重定向至 `/map`）—— 核心页面

**功能描述**：系统核心页面，展示交互式中国地图及旅行统计数据。

**模块构成**：

| 模块 | 说明 |
|------|------|
| **顶部统计栏** | 3 个玻璃态卡片：已点亮省份 / 34、已点亮景区 / 总数、点亮率 % |
| **中国地图** | ECharts 渲染，34 个省份。未点亮 = 浅灰色，已点亮 = 绿色。省份名称 hover 时显示 |
| **地图水印** | 右下角 subtle 水印："识界 🌍 light your life"（截图时自动带上品牌） |
| **悬浮侧边栏** | 右侧浮层，显示"✨ 点亮进度 TOP5"（点亮率最高的 5 个省份）。玻璃态半透明背景。支持展开/收起（点击 ✕ 收起为 ✨ 小按钮） |
| **省份交互** | ① **单击** — 侧边栏自动隐藏，地图放大到该省份（平滑动画 700ms），侧边栏显示该省份进度详情，省份边框高亮（琥珀色）<br>② **再次单击** — 恢复全图视角<br>③ **双击** — 进入该省份景区列表页 |
| **匿名态** | 未登录可浏览地图，底部提示"登录后可点亮景区、解锁成就" |

**设计要点**：
- 地图区域是截图分享的核心画面，需要足够美观、深色背景与绿色高亮对比强烈
- 侧边栏收起后地图应完全无遮挡
- 悬浮侧边栏需要高度自适应（根据内容），避免大量留白

#### 页面 2：省份景区详情页（`/province/:id`）

**功能描述**：展示某省份下的所有景区，支持搜索、筛选、点亮/取消、多次记录。

**模块构成**：

| 模块 | 说明 |
|------|------|
| **省份头部** | 返回按钮 + 省份名 + 统计：已点亮 x/总景区（总访问 y 次） |
| **筛选栏** | 名称搜索框 + 分类下拉 + 等级下拉（5A/4A） |
| **批量操作栏** | "全选未点亮" / "取消选择" / "确认点亮 (n)" |
| **景区卡片网格** | 按 5A/4A 分组展示。每个卡片包含：等级标签、分类标签、景区名、点亮次数、每次点亮日期标签 |
| **日期选择弹窗** | 点击"点亮"后弹出 📅 日期选择器，默认当天，可修改历史日期 |
| **卡片状态** | ① **未点亮**：复选框"未点亮/待点亮"<br>② **已点亮**：显示"✨ 已点亮 n 次"+日期标签，提供"+ 再次点亮"和"取消"按钮 |

**设计要点**：
- 日期标签需要清晰可读但不过于抢眼
- "再次点亮"和"取消"按钮需要明显区分
- 弹窗需要移动端友好的触控区域（≥ 44px）

**模块构成**：

|------|------|

**设计要点**：

## 四、核心交互流程

### 4.1 首次点亮一个景区

```
用户进入 /map → 双击省份 → 进入 /province/:id
  → 勾选景区 / 全选未点亮 → 点击"确认点亮"
  → 弹出日期选择弹窗 → 选择日期 → 确认
  → 景区卡片更新为"✨ 已点亮 1 次" + 日期标签
  → 返回 /map 该省份变为绿色
  → 若满足成就条件，自动弹出 Toast 提示解锁成就
```

### 4.2 再次点亮同一景区

```
用户进入 /province/:id → 点击已点亮景区的"+ 再次点亮"
  → 弹出日期选择弹窗 → 选择不同日期（如 2019-03-16）
  → 确认 → 卡片更新为"✨ 已点亮 2 次" + 两个日期标签
```

### 4.3 截图分享场景

```
用户进入 /map → 收起侧边栏（✕）→ 地图完全无遮挡
  → 截图 → 截图自动包含右下角水印"识界 🌍 light your life"
```

## 五、当前 UI 现状（已上线版本参考）

### 5.1 已实现的视觉风格

| 维度 | 当前状态 |
|------|---------|
| **主色调** | 蓝色系（#2563eb 为主，#3b82f6 为辅） |
| **背景** | 地图页：蓝灰渐变（#eef2f6 → #e2e8f0 → #f0f4f8）；其他页：白色 |
| **卡片** | 白色圆角卡片 + 微妙阴影 |
| **玻璃态** | 统计栏、侧边栏使用 backdrop-filter: blur + 半透明背景 |
| **地图颜色** | 未点亮：#e2e8f0（浅灰）；已点亮：#22c55e（绿色）；高亮：#f59e0b（琥珀边框） |
| **字体** | -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Noto Sans SC', sans-serif |
| **圆角** | 卡片 12-16px，按钮 8-12px |
| **动画** | 地图放大 700ms cubicOut，侧边栏滑入 350ms，进度条 500ms ease |

### 5.2 已知 UI 问题（设计团队可重点优化）

1. **整体风格偏工具化，缺少情感温度** — 当前更像一个数据管理工具，缺乏旅行产品的浪漫感和探索欲
2. **地图截图不够"惊艳"** — 虽然已改为绿色高亮+渐变背景，但和专业的旅行/足迹类产品相比，截图的视觉冲击力仍有提升空间
3. **成就徽章设计粗糙** — 当前使用 emoji 字符串，需要设计统一的矢量徽章体系
4. **移动端侧边栏体验** — 悬浮面板在小屏幕上仍可能遮挡地图关键区域
5. **空态/缺省页设计缺失** — 未登录态、无数据态的视觉表现力不足
6. **无品牌插画/图形系统** — 缺乏统一的品牌插画风格来强化"识界"的认知

## 六、给 UI 设计团队的设计建议

### 6.1 风格方向建议（供参考，非定稿）

我们倾向探索以下几个方向，欢迎设计团队提案：

| 方向 | 描述 | 参考气质 |
|------|------|---------|
| **A. 温暖手绘风** | 手绘地图质感、水彩风格的省份填充、手写字体 | 小红书、旅行手账 |
| **B. 暗夜星图风** | 深色背景（深蓝/深灰）、已点亮省份像星星一样发光、星座连线效果 | Star Walk、Sky Guide |
| **C. 极简杂志风** | 大量留白、精致排版、摄影感强的配图、优雅字体 | Cereal 杂志、Kinfolk |
| **D. 现代国潮风** | 中国传统色（朱砂、石青、藤黄）、水墨质感、书法字体 | 故宫文创、站酷国潮 |

### 6.2 必须满足的设计约束

1. **截图友好**：地图首页截图必须好看，因为用户最核心的分享场景是截图发朋友圈/小红书
2. **移动端优先**：60% 以上用户通过手机访问，所有交互必须 touch-friendly（最小触控目标 44px）
3. **品牌水印可见**：右下角水印"识界 🌍 light your life"在截图中需要清晰但不过于抢眼
4. **色盲友好**：地图高亮不能仅依赖绿色，需要有纹理/图案等辅助区分
5. **深色模式预留**：建议设计时考虑后续深色模式的扩展性

### 6.3 需要设计团队输出的交付物

| 优先级 | 交付物 | 说明 |
|--------|--------|------|
| P0 | 品牌视觉系统 | Logo（识界 + 🌍 图形）、品牌色板、字体规范、品牌插画风格 |
| P0 | 地图首页视觉 | 地图容器样式、省份颜色体系（未点亮/已点亮/悬停/选中）、统计栏样式、侧边栏样式 |
| P0 | 移动端适配方案 | 底部 Tab 栏、悬浮面板、弹窗、触控区域优化 |
| P1 | 省份详情页 | 景区卡片样式、筛选栏、批量操作栏、日期选择弹窗、日期标签 |
| P1 | 成就徽章系统 | 23 个成就的统一徽章设计（unlocked/locked 状态） |
| P1 | 旅程页 | 访问列表卡片、统计头部、分组样式 |
| P2 | 登录页 | 背景装饰、玻璃态卡片、表单样式 |
| P2 | 管理后台 | 侧边栏、表格、表单 |
| P2 | 空态/缺省页 | 未登录、无数据、加载中、错误页 |

## 七、技术约束

| 项目 | 说明 |
|------|------|
| 前端框架 | React 19 + TypeScript + Vite |
| 样式方案 | 纯 CSS（无 Tailwind/Styled-components），设计团队可直接提供 CSS |
| 地图引擎 | ECharts（GeoJSON 格式），省份颜色可通过 ECharts itemStyle 动态配置 |
| 图标方案 | 当前使用 emoji，建议设计团队提供 SVG 图标集替换 |
| 响应式断点 | 桌面端 ≥ 769px，移动端 ≤ 768px |
| 支持浏览器 | Chrome/Safari/Firefox 最新 2 个版本，iOS Safari 14+ |

## 八、联系方式与协作方式

- **前端开发者**：可根据设计稿直接修改 CSS/React 组件
- **数据对接**：所有页面数据已通过 API 对接完成，UI 调整不涉及后端改动
- **迭代方式**：建议设计团队先输出地图首页（P0 最高优先级）的视觉方案，确认方向后再铺开其他页面
