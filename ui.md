# 「识界」App UI 视觉设计规范

## 1. 产品基础信息

**App 名称：** 识界
**Slogan：** Light your life
**产品类型：** 旅行记录系统 / 地图点亮打卡系统

「识界」是一款以地图点亮为核心的旅行记录 App。用户可以通过景区、景点、城市及旅行路线打卡，逐步点亮自己的旅行地图，形成可视化旅行成就，并激发持续探索世界的欲望。

---

## 2. 本次视觉改版目标

本次需要对「识界」整体视觉进行升级，目标是打造一个更具时尚感、自然感、探索感和社交分享感的浅色主题。

重点优化方向：

1. 色彩不再以纯色为主。
2. 视觉需要体现蓝天、海洋、热带雨林、山林、雪山等自然旅行元素。
3. 页面需要保持浅色、清透，但不能平淡。
4. 地图点亮、路线打卡、成就徽章需要有明显的发光感和仪式感。
5. 分享海报页需要具备社交传播价值，让用户愿意截图分享到小红书、朋友圈、Instagram 等平台。
6. 整体风格需要年轻、时尚、有设计感，而不是普通工具型地图 App。

## 4. 不希望出现的风格

请避免以下设计方向：

* 纯黄色主视觉
* 传统旅游 App 风格
* 普通地图导航 App 风格
* 过于平淡的浅色 UI
* 大面积高饱和色块
* 过度卡通化、幼态化
* 过度游戏化的成就系统
* 普通白底卡片堆叠，缺少层次感

---

## 5. 整体视觉方向

整体采用：

> 自然渐变 + 清透玻璃卡片 + 地图光点 + 高级留白 + 发光路线

整体基调为浅色，但需要通过渐变、光晕、模糊玻璃卡片、地图纹理和风景图片增强层次。

核心视觉符号是 **「点亮」**。

因此，地图点、打卡点、旅行路线、成就徽章、进度条、分享海报都需要围绕以下感受设计：

* 光点扩散
* 路线发光
* 地图区域逐步被点亮
* 成就被解锁
* 用户的旅行世界正在变亮

## 8. 字体规范

### 8.1 中文字体

优先使用系统字体：

```css
font-family: PingFang SC, HarmonyOS Sans, Noto Sans SC, Microsoft YaHei, sans-serif;
```

### 8.2 英文 Slogan

“Light your life” 建议使用更轻盈、旅行感的斜体或手写感字体，但不要过度花哨。

推荐方向：

* Inter Italic
* SF Pro Display Italic


## 11. 玻璃态卡片规范

页面卡片建议使用清透玻璃态，而不是普通白底卡片。

```css
background: rgba(255, 255, 255, 0.78);
backdrop-filter: blur(20px);
border: 1px solid rgba(255, 255, 255, 0.72);
box-shadow: 0 12px 32px rgba(22, 135, 232, 0.10);
border-radius: 24px;
```

适用场景：

* 首页地图进度卡片
* 景点详情信息卡
* 路线记录卡
* 成就数据卡
* 分享海报数据浮层
* 底部导航栏

## 13. 地图视觉规范

地图是产品核心，需要重点强化“点亮世界”的视觉表达。

### 13.1 地图底图

地图底图不要使用强烈灰色道路底图，而是使用浅色自然地图风格。

推荐表现：

| 区域    | 视觉表现    |
| ----- | ------- |
| 海洋    | 浅蓝渐变    |
| 山地    | 淡蓝灰     |
| 森林    | 淡绿色     |
| 雪山    | 冷白 + 淡蓝 |
| 已点亮区域 | 蓝绿渐变覆盖  |
| 未点亮区域 | 低饱和浅灰蓝  |

---

### 13.2 打卡点

打卡点不要使用普通地图 Pin 样式，而是设计成“发光坐标点”。

| 状态   | 样式           |
| ---- | ------------ |
| 未点亮  | 浅灰蓝描边点       |
| 已点亮  | 蓝绿发光点        |
| 当前推荐 | 蓝色外圈呼吸光效     |
| 成就地点 | 蓝绿点 + 少量金色星芒 |

推荐样式：

```css
.checkin-point {
  background: #12C7D9;
  box-shadow: 0 0 24px rgba(18, 199, 217, 0.32);
}
```

---

### 13.3 路线

路线需要使用渐变发光线，不要使用普通实线。

推荐样式：

```css
background: linear-gradient(90deg, #1687E8 0%, #12C7D9 50%, #35C98B 100%);
box-shadow: 0 0 18px rgba(18, 199, 217, 0.24);
```

路线表现建议：

* 线宽：3–5px
* 起点、终点需要强化
* 关键节点可以叠加风景缩略图
* 已完成路线可更亮
* 未完成路线可用低透明虚线

## 14. 页面设计要求

## 14.1 首页 / 地图页

### 页面目标

让用户一进入 App 就感受到：

> 我的世界正在被点亮。

### 页面元素

* 顶部展示品牌「识界」与 slogan
* 主体为中国 / 世界地图
* 已点亮城市以发光点展示
* 已点亮区域以蓝绿渐变覆盖
* 底部浮层展示旅行进度
* 底部悬浮式导航栏

### 核心数据

* 点亮中国 / 世界进度
* 已点亮城市数
* 打卡景点数
* 旅行天数
* 总里程

### 视觉要求

* 地图需要有轻微空间感
* 卡片悬浮在地图上方
* 主进度条使用蓝绿渐变
* 已点亮点位需要有光晕
* 底部导航采用半透明玻璃态
* 首页整体需要有强烈的探索感和成就感

---

## 14.2 景点详情页

### 页面目标

让景点详情页既像旅行卡片，也像收藏级旅行记录。

### 页面结构

* 顶部大图
* 景点名称
* 地理位置
* 景点标签
* 点亮进度
* 已打卡景点横向列表
* 景点介绍
* 底部主按钮「立即打卡」

### 视觉要求

* 顶部图片需要有旅行杂志感
* 图片底部可加入柔和渐变遮罩
* 标签使用浅蓝、浅绿、浅灰胶囊样式
* 打卡按钮使用蓝绿渐变
* 已点亮状态使用小型发光徽章
* 页面不能像普通表单页，需要有情绪感和收藏感

---

## 14.3 路线记录页

### 页面目标

突出“我的旅行轨迹”和“这段旅程值得被记录”。

### 页面结构

* 顶部旅程卡片
* 路线地图
* 行程时间线
* 关键节点 / 城市 / 景点
* 总里程、天数、打卡数

### 视觉要求

* 路线图使用蓝绿渐变发光线
* 节点使用小型风景缩略图或发光圆点
* 时间线使用细线 + 圆点
* 当前行程可使用轻微动态光效
* 信息层级清晰，避免堆积

---

## 14.4 我的成就页

### 页面目标

让用户有成长感、收集感和炫耀感。

### 页面结构

* 用户头像
* 旅行者等级
* 经验进度条
* 已点亮城市地图缩略图
* 旅行足迹数据
* 成就徽章
* 连续打卡
* 分享入口

### 视觉要求

* 成就徽章要有高级感
* 徽章可融合晶体、雪山、海洋、森林等元素
* 已完成徽章使用蓝绿渐变 + 少量金色高光
* 未完成徽章使用低饱和灰蓝
* 页面顶部个人成就区域需要适合截图分享

---

## 14.5 分享海报页

### 页面目标

分享海报页是本次视觉升级重点。

目标是让用户愿意主动截图或一键分享到社交平台。

### 海报内容

* 品牌「识界」
* Slogan：Light your life
* 用户名
* 本次旅行名称
* 精选风景图
* 路线发光轨迹
* 旅行天数
* 总里程
* 打卡景点数
* 一句旅行文案
* 二维码 / App 标识

### 视觉要求

* 竖版比例：3:4 或 9:16
* 大图占比：45%–60%
* 数据区使用半透明玻璃卡片
* 背景使用天空蓝到雪山白渐变
* 路线轨迹可以叠加在风景图上
* 路线需要有发光感
* 底部保留品牌露出
* 整体质感参考小红书 / Instagram 旅行分享卡片
* 需要像一张精致的旅行纪念卡，而不是普通数据截图

## 16. 成就徽章规范

徽章需要避免廉价游戏感，整体应该更像高级旅行勋章。

### 16.1 徽章分类

* 海洋探索者
* 山林漫游者
* 雪山征服者
* 城市点亮者
* 路线记录者
* 世界收藏家

### 16.2 徽章视觉

已完成徽章：

* 蓝绿渐变
* 晶体质感
* 少量金色高光
* 轻微立体感
* 边缘有柔和光晕

未完成徽章：

* 灰蓝色
* 低透明度
* 无发光
* 保持轮廓但降低存在感

## 17. 动效规范

动效需要轻盈、克制，重点突出“点亮”的仪式感。

### 17.1 推荐动效

| 场景     | 动效                |
| ------ | ----------------- |
| 打卡成功   | 光点从当前位置扩散，地图区域被点亮 |
| 路线生成   | 路线从起点向终点流动绘制      |
| 成就解锁   | 徽章轻微放大，光芒扫过       |
| 首页地图   | 已点亮点位轻微呼吸         |
| 分享海报生成 | 风景图淡入，路线轨迹渐显      |

### 17.2 动效时长

| 类型     |         时长 |
| ------ | ---------: |
| 普通点击反馈 |  200–300ms |
| 页面切换   |  300–450ms |
| 点亮仪式   |  600–900ms |
| 成就解锁   | 700–1000ms |

## 19. 页面优先级建议

如果需要分阶段改版，建议 Code X 优先处理以下模块：

### 第一优先级

1. 首页 / 地图页
2. 分享海报页
3. 主色彩与 Design Tokens

原因：这三部分最能快速改变整体视觉感知。

### 第二优先级

1. 景点详情页
2. 路线记录页
3. 底部导航栏

原因：这三部分决定用户记录和打卡的核心体验。

### 第三优先级

1. 我的成就页
2. 成就徽章系统
3. 动效与细节光效

原因：这三部分决定长期留存和分享动力。

## 20. 验收标准

改版后的界面需要满足以下标准：

* 第一眼能感受到旅行、自然、探索，而不是工具型地图。
* 色彩不再以黄色为主，而是以蓝、青、绿、雪白为核心。
* 页面是浅色主题，但具有明显层次，不显得空、不显得淡。
* 首页地图点亮效果明显，有成就感。
* 路线记录页有轨迹感和故事感。
* 成就页有收集欲和成长感。
* 分享海报页具备社交传播价值，截图后像精美旅行卡片。
* 地图点、路线、徽章、按钮都能体现“Light your life”的光感主题。
* 整体视觉年轻、时尚、清透、有高级感。
* 避免过度黄色、过度平面、过度工具化。
