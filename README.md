# 旅行足迹地图系统 🌍

> 用足迹丈量中国 —— V0.2.1

## 项目简介

一个聚焦中国境内的旅行记录系统，核心以**中国地图点亮**为载体，通过标记 4A/5A 级景区帮助用户可视化旅行成就，激发探索欲望。

## 功能特性

### 地图展示与统计
- 🗺️ **交互式中国地图** — 基于 ECharts + GeoJSON 渲染 34 个省级行政区
- ✨ **省份点亮** — 一个省份内只要有一个景区被点亮，该省在地图上显示为绿色
- 📊 **实时统计** — 已点亮省份数、已点亮景区数、点亮率

### 省份景区浏览与点亮
- 🏔️ **景区列表** — 按 5A/4A 分组展示，组内按名称排序
- 🔍 **搜索与筛选** — 支持景区名称模糊搜索、分类筛选、等级筛选
- ✅ **批量点亮** — 多选景区后一键确认点亮

### 用户系统
- 👤 **注册/登录** — 用户名+密码，数据与账号绑定
- 🔒 **登录态保持** — JWT Token 本地存储

### 找回我的足迹
- 🧭 **首次引导** — 新用户可通过四步流程找回历史旅行足迹
- 🏙️ **城市检索** — 支持热门城市、按省份浏览及关键词搜索
- 📍 **批量补录** — 可跨城市选择景区并批量写入访问记录
- 🕒 **模糊时间** — 支持具体日期、年月、年份、季节及待补充时间
- ✨ **结果反馈** — 展示本次新增、累计点亮和新解锁成就

### 成就与进度系统
- 📈 **省份点亮进度** — 环形图按地理大区展示
- 🏷️ **分类点亮进度** — 横向进度条展示各分类完成情况
- 🏅 **成就墙** — 省份探索线(5级) + 景区达人线(10级) + 特殊彩蛋成就(8个)

## 技术栈

| 层 | 技术 |
|---|---|
| 前端 | React 19 + TypeScript + Vite + ECharts + React Router |
| 后端 | Node.js + Express 5 + TypeScript + better-sqlite3 |
| 地图 | 阿里云 DataV GeoJSON |

## 快速开始

### 1. 启动后端

```bash
cd server
npm install
npx tsx src/seed.ts    # 初始化数据（仅需执行一次）
npx tsx src/index.ts   # 启动服务（默认 3001 端口）
```

### 2. 启动前端（开发模式）

```bash
cd client
npm install
npm run dev            # 默认 5173 端口，已配置代理到后端
```

### 3. 生产部署

```bash
cd client && npm run build    # 构建前端到 client/dist
cd server && npx tsx src/index.ts   # 后端会自动 serve client/dist
```

### 4. 测试规范

> **硬性要求：每次代码修改或功能编码后，必须执行一轮端到端测试，确保功能无误、与需求不冲突，方可交付。**

测试 checklist：
1. 前端构建通过：`cd client && npm run build`
2. 后端构建通过：`cd server && npm run build`
3. 核心功能验证（根据修改范围选择）：
   - 用户注册 / 登录 / JWT 持久化
   - 地图省份颜色渲染（点亮后变绿）
   - 景区批量点亮 / 取消点亮
   - 省份 TOP5 侧边栏交互（单击显示进度、双击进入列表）
   - 成就系统自动解锁与 Toast 提示
   - 管理后台权限控制
4. 移动端验证：通过局域网 IP 访问，确认触摸交互正常
5. 服务重启验证：`node server/dist/index.js`，确认静态资源与 API 均正常响应

## 默认数据

- **省份**: 34 个省级行政区（含港澳台）
- **城市节点**: 347 个（含直辖市、港澳台及特殊行政区）
- **景区**: 869 个4A/5A级景区（覆盖全部省级行政区）
- **分类**: 11 个预置分类
- **成就**: 省份、城市、景区、分类、收藏及特殊成就体系

## 版本管理规范

### 分支策略

| 分支 | 用途 | 规则 |
|---|---|---|
| `main` | 正式版本（当前 V0.2.1） | **仅存放稳定版本**，禁止直接提交 |
| `develop` | 开发版本 | 所有日常开发、功能迭代在此分支进行 |
| `hotfix/*` | 紧急修复 | 从 `main` 切出，修复后合并回 `main` 和 `develop` |

### 版本发布流程

1. **日常开发**：所有新功能、优化在 `develop` 分支上迭代
2. **测试验证**：每次修改后按「测试规范」执行端到端测试
3. **用户确认**：开发完成后，由用户**主动提出**"将当前开发版本发布为正式版"
4. **合并发布**：将 `develop` 合并到 `main`，并打上版本 tag（如 `v0.2.1`）
5. **禁止擅自覆盖**：**未经用户明确要求，不得将 develop 的修改覆盖到 main**

### 当前版本

- **正式版**: `v0.2.1` — 新增“找回我的足迹”引导与批量补录流程，并完善城市数据和旅行时间记录
- **开发版**: `develop` 分支 — 持续迭代中
